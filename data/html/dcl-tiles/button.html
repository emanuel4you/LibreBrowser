button

DCL Tiles

: button

Create a button may or may not include a label.

DCL Active Tile

This active tile facilitates the creation of a button, which can feature a label or remain unlabelled. Typically, a button serves the purpose of closing a dialog box or presenting an additional box (referred to as "tunneling dialog boxes"). It is essential for every dialog box to contain at least one button linked to the [link]LISP function [link]done_dialog; failure to do so will result in the program becoming unresponsive.

: button {

action = "(done_dialog)";

label = "OK";

}

NOTE

Several predefined tiles are available that encompass the most frequently used button configurations, such as the ok_only attribute, ok_cancel, and ok_cancel_help tiles.

Valid Attributes

: button {

action = "(string)";

alignment = position;

fixed_height = flag;

fixed_width = flag;

height = number;

is_cancel = flag;

is_default = flag;

is_enabled = flag;

is_tab_stop = flag;

key = "string";

label = "string";

mnemonic = "char";

width = number;

}

EXAMPLE

: button {

action = "(convdxf)";

label="Convert to DXF";

mnemonic="C";

key="BC";

}

NOTE

The label attribute indicates the text on the button-for

EXAMPLE

label = "Sync Grid to Snap"
